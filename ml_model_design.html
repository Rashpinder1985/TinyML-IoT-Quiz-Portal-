Designing ML models for embedded systems means balancing accuracy, latency, energy efficiency, and memory footprint.

üîß 1. Constraints of Embedded Systems

  • Limited RAM (often < 256 KB)
  • Limited Flash/storage space
  • Low clock speeds (MHz, not GHz)
  • Battery-powered (low energy budgets)

üéØ 2. Design Strategies

  • Use small models: e.g., MobileNet, TinyCNN, decision trees
  • Layer optimization: Fewer layers, smaller kernels, fewer filters
  • Use integer operations: Favor quantized (int8) over float32
  • Early exit architectures: Allow fast inference on easy inputs

üì¶ 3. Tools That Help

  • TensorFlow Lite for Microcontrollers (TFLM)
  • CMSIS-NN for ARM Cortex-M optimization
  • uTensor / TVM for custom compilation

Designing a gesture recognition model for a microcontroller:

  • Use 1D CNN with 8-bit quantization
  • Limit total parameters < 20k
  • Target RAM: ~64KB
