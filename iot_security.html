This module provides a foundational understanding of the challenges and countermeasures related to security, privacy, and ethics in Internet of Things (IoT) ecosystems.

üîì Common Attack Vectors (Explained)

  • Physical attacks: Involve tampering with IoT hardware (e.g., removing memory cards or replacing firmware) to bypass built-in security mechanisms.
  • Malware: Specially crafted malicious software, like the Mirai botnet, which hijacks poorly secured IoT devices and uses them in large-scale attacks (e.g., DDoS).
  • Credential abuse: Happens when attackers exploit weak or reused passwords to gain unauthorized access to devices.
  • Man-in-the-Middle (MitM) attacks: Occur when communication between two devices is intercepted by an unauthorized third party, especially if the data is not encrypted.

üìå Case Study: NightWatch Camera Breach

Smart surveillance cameras branded "NightWatch" were compromised, exposing live video feeds to hackers. This breach was caused by:

  • Use of default or weak passwords without user prompts to change them
  • Absence of TLS encryption, allowing attackers to intercept video feeds
  • Lack of a proper software update mechanism, preventing timely patching of vulnerabilities

Mitigations included:

  • Enabling TLS encryption to secure data-in-transit
  • Integrating decentralized AI to process sensitive data locally, reducing exposure
  • Deploying anomaly detection to identify unusual device behavior in real-time

üîê Key Security Techniques

  • RSA: A public-key encryption scheme often used for secure key exchange but relatively heavy for IoT.
  • AES: A symmetric key algorithm ideal for fast and efficient bulk data encryption.
  • ECC (Elliptic Curve Cryptography): Offers strong security with smaller keys, making it suitable for memory-constrained IoT devices.
  • Authentication methods:
    • One-way: Only the client verifies the server (e.g., HTTPS)
    • Mutual: Both ends verify each other
    • Centralized: One entity manages credentials (e.g., cloud-based login)
    • Distributed: Peer-to-peer or blockchain-based validation
  • PKI (Public Key Infrastructure): A framework that manages digital certificates and public keys, ensuring secure data exchange and identity verification.

üß≠ Ethical Design Principles

  • Transparency: Clearly inform users about what data is being collected and why.
  • Fairness: Ensure AI decisions are unbiased and inclusive (e.g., avoiding racial/gender bias in smart surveillance).
  • Accountability: Define who is responsible for outcomes caused by automated decisions or data leaks.
  • Opt-out: Allow users to disable tracking features or data sharing without sacrificing access to core functionality.
